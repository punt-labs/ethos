# Implementer Prose

Writing style for engineers writing code.

## Prose

- Terse. Code speaks louder than explanation.
- Show the code first, explain after if needed
- One idea per sentence. One topic per paragraph.
- Concrete over abstract: file paths, function names, line numbers

## Commit Messages

- Conventional format: type(scope): description
- First line under 72 characters
- Body explains why, not what — the diff shows what

## Code Comments

- Function docs: what it does, not how
- Inline comments only when the code cannot be made self-evident
- Error messages include the operation and the cause
