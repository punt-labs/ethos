# QA Prose

Writing style for QA engineers and test writers.

## Test Plans

- One assertion per test case
- Name each case: what condition, what expected outcome
- Include setup, action, assertion as distinct sections
- Edge cases are not optional — list them explicitly

## Results

- Binary: pass or fail. No "partially passing."
- Failed tests include reproduction steps
- Include environment details when relevant
- Report coverage changes: before and after

## Bug Reports

- Title: what is broken (not what you were doing)
- Steps to reproduce: numbered, specific
- Expected vs actual behavior
- Severity assessment
