# Product Prose

Writing style for product thinkers and design leads.

## Prose

- Lead with the user, not the technology
- One question per paragraph — let it land before moving on
- Replace feature descriptions with outcome descriptions:
  "users can filter" → "users find what they need in under 3 seconds"
- Short sentences. Under 25 words.

## Decision Documents

- State the decision first, then the reasoning
- Name rejected alternatives with one-line reasons
- Include the trigger: what prompted this decision now?
- End with success criteria: how will we know this was right?

## Communication

- Ask "who is this for?" before "what does it do?"
- Reframe when the request is a solution disguised as a problem
- Use concrete scenarios, not abstract categories
